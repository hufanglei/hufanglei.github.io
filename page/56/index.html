maven项目打包跳过测试

发表于 2018-12-11 更新于 2025-01-24 分类于 maven 本文字数： 499 阅读时长 ≈ 1 分钟

运行mvn install时跳过Test

方法一:

	<project>
	  [...]
	  <build>
	    <plugins>
	      <plugin>
	        <groupId>org.apache.maven.plugins</groupId>
	        <artifactId>maven-surefire-plugin</artifactId>
	        <version>2.18.1</version>
	        <configuration>
	          <skipTests>true</skipTests>
	        </configuration>
	      </plugin>
	    </plugins>
	  </build>
	  [...]
	</project>

方法二:

mvn install -DskipTests

或者

mvn install -Dmaven.test.skip=true

linux下安装和卸载自己想要版本的git版本

发表于 2018-12-11 更新于 2025-01-24 分类于 linux git 本文字数： 844 阅读时长 ≈ 1 分钟

1.Linux下安装

yum install curl-devel expat-devel gettext-devel openssl-devel zlib-devel gcc perl-ExtUtils-MakeMaker

2.删除已有的git

yum remove git

3.下载git源码

  • //切换到你的包文件存放目录下

cd /usr/src

  • //下载git安装包

wget https://github.com/git/git/archive/v2.9.2.tar.gz

实在下载不了，可以在[我的网盘](https://pan.baidu.com/s/1caDt-tj5cy1A_rj0H4kldg ）提取码：r9en 下载)

  • //解压git安装包
    tar -xzvf v2.9.2.tar.gz -C ~/training/
    cd git-2.9.2
  • //配置git安装路径

make prefix=/usr/local/git all

  • //编译并且安装

make && make install

4.将git指令添加到环境变量bash中

echo “export PATH=$PATH:/usr/local/git/bin” >> /etc/bashrc

实时生效

source /etc/bashrc

查看版本号

git –version

git version 2.9.2

git如何卸载

1.找到git的位置

which -a git

2.进入git所在的目录

cd /usr/bin/git(这个是一般的默认位置)

3.删除命令

sudo rm -rf git

linux下redis的安装和配置

发表于 2018-12-10 更新于 2025-01-24 分类于 linux 大数据 本文字数： 935 阅读时长 ≈ 1 分钟

1.官网下载redis

2.用ftp传到服务器上

3.解压redis

tar -zxvf redis-3.0.5.tar.gz

4.编译安装到自定义目录下（我的/root/training/redis）

cd /root/tools/edis-3.0.5
[root@localhost redis-3.0.5]# make
（如果报错jemalloc/jemalloc.h：没有那个文件或目录，试试make MALLOC=libc）
（如果报错，说明没有安装gcc，必须先执行这个命令: yum install -y gcc g++ gcc-c++ make ）
[root@localhost redis-3.0.5]# make PREFIX=/root/training/redis install

5.进入到自定义的redis工作目录下

[root@localhost training]# cd redis/

6. 新建conf文件夹

[root@localhost redis]# mkdir conf

7.拷贝源文件中的conf到步骤6的conf下

[root@localhost redis]# cp ~/tools/redis-3.0.5/redis.conf conf/

8.修改redis.conf的42行改为后台运行，默认是否

daemonize yes

9.运行redis

/root/training/redis/bin/redis-server /root/training/redis/conf/redis.conf

10.修改redis为自启动

vi /etc/rc.local
//添加下面的内容
/root/training/redis/bin/redis-server /root/training/redis/conf/redis.conf

自此，大功告成!!

ionic3 左侧菜单栏显示 用户名

发表于 2018-11-26 更新于 2025-01-24 本文字数： 4.5k 阅读时长 ≈ 4 分钟

分析

  • ionic3导航统一用的是nav导航，可以携带参数，但是左侧菜单栏是在app的根组件中，他的生命周期中，只会加载一次，故，退出登录后，app里面的变量属性不会切换。这里就造成了 用户名无法切换的问题，只显示第一次加载的值

解决办法

  • 参看了下ionic3的官方，可知，有个events的对象，可以发布和订阅消息，这个知识点可以解决我们上面的问题，在app加载的时候 订阅用户的信息。在登录后发布新的用户信息。
