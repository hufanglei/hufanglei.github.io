ssh拷贝文件

发表于 2023-02-28 更新于 2025-01-24 分类于 运维 本文字数： 433 阅读时长 ≈ 1 分钟

@TOC

前言

在linux上部署项目，经常用到两个服务器之间传输文件，如果从一个服务器下载到本地再传上另一个服务器上，一是很繁琐，二是浪费时间。常规操作是使用ssh拷贝文件进行传输。

服务器A： 在有源文件的服务器A上生成密钥对

执行 ssh-keygen -t rsa 可以一路回车键。最后生成2个文件。

服务器B(目标服务器)

手动创建: .ssh/authorized_keys 文件。
将服务器A的公钥复制到authorized_keys内容中，记得这里最好用追加;

拷贝文件

在服务器A上把想传递的文件拷贝到服务器B上，可以执行下面的命令:
scp java-admin.jar root@172.16.10.239:/home/nvidia/tf_software/deploy/

这样就把java-admin.jar 拷贝到了服务器B（172.16.10.239)的deploy目录下了。
